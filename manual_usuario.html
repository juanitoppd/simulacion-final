Manual de Usuario - Simulacion TechClassUC

Version: 1.0

Audiencia: Estudiantes de simulacion y modelos de colas

Requisito previo: Python instalado y terminal PowerShell

1. Requisitos

  • Python instalado en Windows.
  • PowerShell abierto en la carpeta del proyecto.
  • Dependencias del archivo requirements.txt.
  • Navegador web para abrir la vista de resultados.

2. Instalacion y primer arranque

Ubicate en la carpeta del proyecto:

cd "C:\Users\JUAN CAMILO\Downloads\simulacion final"

Instala las librerias necesarias:

py -m pip install -r requirements.txt

3. Ejecutar la simulacion

Para correr el programa con los valores por defecto:

py main.py

Para hacer una prueba rapida:

py main.py --replicas 3 --replicas-sensibilidad 2 --tiempo 120 --warmup 10 --salida resultados_prueba_terminal

La ejecucion genera imagenes PNG y un archivo CSV con los resultados de sensibilidad.

4. Usar la vista web

Abre la pagina local de resultados:

vista_resultados.html

Desde esa vista puedes revisar las graficas y entrar a este manual usando el boton superior.

Ir a la vista de resultados

5. Interpretar las graficas

Evolucion del sistema

Muestra como cambia el numero de clientes en el sistema y en cola durante el tiempo simulado.

Histograma de esperas

Resume la frecuencia de los tiempos de espera antes de recibir atencion.

Wq vs tecnicos

Compara el tiempo promedio de espera al cambiar la cantidad de tecnicos disponibles.

Rho vs lambda

Permite revisar el nivel de utilizacion del sistema frente a distintas tasas de llegada.

Heatmap Wq

Ayuda a identificar combinaciones de tasa de llegada y numero de tecnicos con menor espera.

6. Parametros principales

  • --lambda-llegadas: clientes que llegan por hora.
  • --mu-servicio: clientes atendidos por hora por cada tecnico.
  • --servidores: cantidad de tecnicos disponibles.
  • --replicas: cantidad de repeticiones Montecarlo.
  • --salida: carpeta donde se guardan los resultados.

7. Solucion de problemas

No aparecen las graficas

Ejecuta nuevamente la simulacion y verifica que exista la carpeta resultados_prueba_terminal.

Falta una libreria

Instala dependencias otra vez:

py -m pip install -r requirements.txt

El sistema queda inestable

Aumenta el numero de tecnicos o reduce la tasa de llegada para que la utilizacion sea menor que 1.

8. Subir el proyecto a Git

Despues de crear el repositorio remoto, agrega los archivos y realiza el commit:

git add . git commit -m "Agregar simulacion TechClassUC"

Conecta el remoto y sube la rama principal:

git remote add origin URL_DEL_REPOSITORIO git branch -M main git push -u origin main
